Keil MDK 深度调试指南

编写代码只是开发过程的一半，另一半则是调试 (Debug)。Keil MDK 提供了非常强大的调试功能，能够帮助我们深入单片机内部，查看每一个寄存器、每一个变量的实时状态。掌握这些技巧，能让你的 Bug 修复效率提升十倍。

一、进入调试模式

在进行硬件调试前，请确保你的仿真器（ST-Link/J-Link）已连接，并且在 Options for Target -> Debug 中选择了正确的调试器。

点击工具栏上的红色 "d" 图标，或者按下快捷键 Ctrl + F5，即可进入调试模式。

进入后，界面会发生变化，出现寄存器窗口、反汇编窗口等。此时程序默认会在 main() 函数的第一行停下来，等待你的指令。

二、基本调试控制

调试最常用的就是控制程序的运行流程。工具栏左上角有一排控制按钮：

  • Reset (复位)：复位单片机，程序回到起始点。
  • Run (全速运行) F5：程序开始全速执行，直到遇到断点。
  • Stop (停止)：暂停程序运行。
  • Step (单步调试) F11：执行当前行代码。如果当前行是函数调用，会进入函数内部。
  • Step Over (单步跳过) F10：执行当前行代码。如果当前行是函数调用，会直接执行完该函数，不进入内部。这是最常用的单步方式。
  • Step Out (跳出函数)：执行完当前函数剩余的代码，返回到调用该函数的地方。
  • Run to Cursor Line (运行到光标处)：右键点击某一行代码选择此项，程序会运行到该行并暂停。

三、查看变量 (Watch Window)

这是调试中最常用的功能，用于实时查看全局变量或局部变量的值。

1. 添加变量

在代码窗口中，选中你想查看的变量名，右键点击选择 "Add 'xxx' to... -> Watch 1"。 或者直接在下方的 Watch 1 窗口中双击空白处，手动输入变量名。

2. 实时刷新

默认情况下，只有当程序暂停时，变量值才会更新。如果你希望在程序全速运行时也能看到变量变化（需要仿真器支持，如 J-Link 或 ST-Link V2 以上），可以勾选 View -> Periodic Window Update。

3. 修改变量值

在 Watch 窗口中，你不仅可以看，还可以改。单击变量的值，输入新数值并回车，单片机内存中的值就会被立即修改。这在测试临界条件（如模拟传感器超限）时非常有用。

四、查看外设寄存器 (System Viewer)

想知道 GPIO 到底有没有配置成推挽输出？定时器的 ARR 寄存器值对不对？不需要查代码，直接看寄存器。

  1. 点击菜单栏的 Peripherals。
  2. 选择 System Viewer。
  3. 选择你想查看的外设（例如 GPIO -> GPIOA）。

右侧会出现一个窗口，列出该外设的所有寄存器及其当前的配置值。你甚至可以直接在这里勾选/取消勾选某些位，实时控制硬件（例如点亮 LED）。

五、查看内存与内核寄存器

1. Memory Window (内存窗口)

如果你需要查看数组、结构体或者一段连续内存的内容，使用 Memory 窗口。 在地址栏输入变量名（如 &my_buffer）或绝对地址（如 0x20000000），即可查看该地址下的原始数据（Hex 格式）。

2. CPU Registers (内核寄存器)

左侧的 Registers 窗口显示了 ARM 内核寄存器的状态：

  • R0 - R15：通用寄存器。
  • SP (Stack Pointer)：栈指针，指向当前栈顶。
  • LR (Link Register)：连接寄存器，保存函数返回地址。
  • PC (Program Counter)：程序计数器，指向当前正在执行的指令地址。

当程序跑飞（HardFault）时，查看 LR 和 PC 是定位问题的关键。

六、高级技巧 (Call Stack)

Call Stack + Locals 窗口可以显示当前的函数调用关系（调用栈）。

例如：main() -> sensor_init() -> i2c_write()。 如果你在 i2c_write 中打了断点，Call Stack 会清晰地展示这一条调用链。右键点击某个上层函数，选择 Show Caller Code，可以快速跳转到调用它的地方。这对于理清复杂的逻辑流非常有帮助。
